# 目标层（Targets Layer）设计开发规范

## 1. 需求分析

### 1.1 背景与目标

目标层的设计源于天文研究的两大核心需求：

**需求一：观测任务设计**
面向天文观测任务的设计阶段，需要将特定观测目标的参数化模型（涵盖加速、辐射、吸积、并合、瓦解、喷发等物理过程）转换为图像、光谱、光变曲线等可观测模型，以支撑观测策略优化、积分时间计算、仪器配置选择等任务规划活动。

**需求二：观测数据分析**
面向观测数据的科学分析阶段，需要从图像、光谱、光变曲线等高级科学数据产品中提取、拟合观测目标模型参数，建立数据与物理模型之间的定量联系，实现物理参数的约束和反演。

### 1.2 功能需求

基于上述背景，目标层需实现三大核心功能：

**功能一：模型全生命周期管理**
建立观测目标参数化模型的收集、组织和管理体系，包括：
- 统一接收天文用户输入的各类观测目标参数化模型
- 语义解析与歧义消除，确保模型描述的精确性
- 冲突检测机制，识别参数间物理不自洽或逻辑矛盾
- 完整性测试，验证模型是否存在未定义的物理环节或计算断点
- 版本控制与溯源，支持模型迭代历史和兼容性管理

**功能二：高效正向建模与仿真**
从参数化模型高效生成具有足够精度的可观测模型：
- 支持多尺度、多维度的物理模型数值仿真（空间、能谱、时域）
- 确保数值仿真计算精度满足观测模拟需求
- 优化计算性能，支持大规模参数空间和蒙特卡洛模拟
- 输出标准化FITS格式的图像、光谱、光变曲线数据模型，无缝衔接观测量层

**功能三：参数反演与模型检验**
从高级科学数据产品中估计模型参数并验证一致性：
- 支持从FITS标准数据产品进行模型参数提取和拟合
- 实现贝叶斯推断、最大似然估计等多种参数估计方法
- 与输入的设计阶段模型进行定量比较，计算偏离度和一致性指标
- 生成模型-数据对比报告，支撑科学结论的可靠性评估

### 1.3 科学主题与层级化组织

**科学主题（Science Themes）的概念来源**

科学主题这一组织概念具有双重来源：

1. **科学驱动**：在前期科学目标研究阶段，科学用户基于研究兴趣和方法论的自然聚类，自发形成了若干科学主题（如系外行星、多信使天文学、星系宇宙学等），这些主题代表了天文研究的前沿领域和重大问题。

2. **工程驱动**：在 OASIS 软件系统设计过程中，我们认识到若采取完全扁平化的观测目标参数化模型组织方式（即每个任务独立开发所有模型），将导致两大严重问题：
   - **重复开发**：不同任务频繁需要相同的基础模型（如研究系外行星需要恒星模型，研究多恒星系统也需要恒星模型，研究星系与超大质量黑洞互馈同样涉及恒星模型）
   - **冲突风险**：各自开发的同类模型（如恒星大气模型、吸积盘模型）在接口、参数定义、单位制等方面极易产生不一致，造成系统集成时的语义冲突和版本混乱

**层级化组织方案**

为解决上述问题，目标层采用层级化的观测目标模型组织方式，形成从通用到专用的模型继承体系：

```
基础模型（Foundation）
    ↓
科学主题（Themes）
    ↓
研究方向（Directions, Optional）
    ↓
观测任务（Missions）
    ↓
具体目标（Targets）
```

- **基础模型**：包含恒星、吸积、引力束缚态、辐射机制等跨领域通用理论模型，作为所有科学主题的共享基础设施
- **科学主题**：围绕重大科学问题组织的模型集合（如致密天体并合、行星系统形成），继承并扩展基础层模型
- **研究方向**（可选）：在大型科学主题内按方法论或对象类别划分的子领域，用于进一步细化模型管理
- **观测任务**：具体的观测提案或巡天项目，继承主题/方向的模型体系，定义特定的参数配置和观测约束
- **具体目标**：单次观测或仿真的具体天体实例，承载最终的物理参数和观测配置

此架构确保基础模型的单一来源（Single Source of Truth），通过继承和组合实现复用，同时通过命名空间隔离避免不同主题间的模型冲突。

## 2. 核心设计原则

### 2.1 可插拔、开放式设计（输入接口）

目标层采用**完全开放式架构**，支持多科学主题（方向）、观测任务（目标）并行发展，确保系统的可扩展性和灵活性：

- **模块化设计**：每个观测目标作为独立模块，可独立开发、测试和部署
- **插件式注册**：新增观测目标模型通过注册机制自动集成，无需修改核心代码
- **标准接口**：所有观测目标模型遵循统一的接口规范，确保互操作性
- **动态发现**：系统运行时自动发现和加载可用观测目标模型
- **版本管理**：支持模型版本控制和向后兼容

### 2.2 统一输出接口

无论何种目标类型、何种物理模型，都必须能够产生以下三种标准输出，确保与观测量层的无缝对接：

- **图像模型**：二维亮度分布（可包含时间/波长维度）
- **光谱模型**：波长-通量关系（可包含时间维度）
- **光变模型**：时间-通量关系（可包含波长维度）

此外，如果提供了必要的观测目标模型，还可产生并输出扩展的**偏振模型**。

## 3. 架构设计

### 3.1 总体架构

目标层采用三段式架构。

分离关注点：

1. **接口段 (Interface segment)**
   - 提供准确、易用的文本接口（YAML格式），负责接受科学用户输入的观测目标模型，用户可显式的定义模型及参数，也可指定参考文献，并通过大模型生成参数化模型
   - 提供统一的API接口，负责与观测量层交互
   - 接收观测请求参数（坐标、波长、时间等）
   - 返回标准化的图像、光谱、光变数据
   - 实现模型注册和发现机制

2. **模型段 (Model segment)**
   - 核心观测目标模型实现层
   - 包含各类目标物理模型（点源、延展源、活动星系核等）
   - 每个模型独立实现，遵循标准接口
   - 支持模型参数验证和边界检查
   - 该段内部采用五层架构，包含**基础**、**主题**、**方向**、**任务**与**目标**

3. **数据段 (Data segment)**
   - 负责目标输入输出模型的存储、读取和缓存
   - 管理参数化模型（YAML格式，记载物理模型的逻辑及附加信息）、参数表或源表（CSV/TSV格式）的解析和验证
   - 处理FITS文件的读写操作
   - 实现数据缓存机制以提高性能

### 3.2 核心组件

#### 3.2.1 接口段 - 输入组件

**Request Handler / 请求处理器**
- 统一接收观测量层的观测请求，解析坐标、波长、时间等参数
- 负责坐标系转换（ICRS/Galactic/Ecliptic）和格式标准化
- 管理请求生命周期和异步任务状态

**YAML Semantic Parser / YAML语义解析器**
- 解析用户输入的YAML格式参数化模型（显式定义或文献引用）
- 消除语义歧义，利用大模型生成参数化模型
- 支持外部文献引用解析和参数自动补全

**Schema Validator / 模式验证器**
- 验证输入参数的结构完整性和类型正确性
- 检查必要字段是否存在，避免未定义的物理环节
- 验证单位制和物理量纲一致性

#### 3.2.2 模型段 - 管理层组件

**Hierarchy Manager / 层级管理器**
- 维护基础→主题→方向→任务→目标的五层级继承链
- 管理命名空间隔离，避免不同科学主题间模型冲突
- 处理模型版本控制和向后兼容性检查

**Conflict Detector / 冲突检测器**
- 检测参数间物理不自洽（如超光速、负温度、非物理的能谱指数）
- 验证跨层级继承时的单位制一致性和接口兼容性
- 识别同名参数在不同模型中的语义冲突

**Completeness Checker / 完整性验证器**
- 检查计算依赖图中是否存在断点（如依赖的质量损失率未定义）
- 验证模型输出所需的中间物理量是否全部可计算
- 确保从输入参数到可观测量的转换链完整无缺

#### 3.2.3 模型段 - 计算引擎组件

**Physics Engine / 物理计算引擎**
- 执行辐射机制、吸积、并合、瓦解等过程的数值仿真
- 管理多尺度计算（空间/能谱/时域）和自适应网格 refinement
- 提供标准物理常数和单位制转换服务

**Numerical Integrator / 数值积分器**
- 处理光线追踪、辐射转移、轨道演化等数值积分任务
- 支持刚性方程组求解和自适应步长控制

**Parallel Scheduler / 并行调度器**
- 管理蒙特卡洛模拟和多参数空间扫描的并行执行
- 支持CPU/GPU异构计算资源调度和MPI分布式计算
- 实现任务队列管理和负载均衡

**Compute Cache Manager / 计算缓存管理器**
- 基于参数哈希的LRU缓存策略，避免重复数值计算
- 管理内存缓存与磁盘缓存的层级，支持计算结果持久化
- 实现缓存失效策略和版本一致性检查

#### 3.2.4 接口段 - 输出组件

**Output Standardizer / 输出标准化器**
- 确保所有模型输出符合统一格式（图像/光谱/光变/偏振）
- 转换数据为FITS标准格式，添加WCS、时间戳、模型版本等元数据
- 实现与观测量层的数据格式无缝对接

#### 3.2.5 数据段组件

**FITS I/O Manager / FITS输入输出管理器**
- 读取高级科学数据产品（支持功能三的参数反演）
- 处理WCS解析、BINTABLE扩展、头部关键字验证
- 支持大文件分块读写和内存映射（mmap）优化

**Catalog Connector / 源表连接器**
- 连接Gaia、SIMBAD、VizieR等外部天文数据库
- 解析和验证CSV/TSV格式参数表，处理缺失数据和异常值
- 支持本地缓存和离线模式

**Model Repository / 模型仓库**
- 持久化存储参数化模型（YAML）和计算结果（FITS/HD5）
- 实现模型版本控制、分支管理和溯源追踪
- 支持模型共享和协作开发工作流

#### 3.2.6 分析组件（跨段协作）

**Parameter Estimator / 参数估计器**
- 实现贝叶斯推断（MCMC、嵌套采样）、最大似然估计等参数反演方法
- 与模型段协作提供先验分布和似然函数接口
- 支持从FITS数据产品直接提取约束条件

**Consistency Evaluator / 一致性评估器**
- 计算设计阶段模型与观测反演结果的偏离度（残差、χ²、贝叶斯证据比）
- 评估模型参数与输入模型的统计一致性
- 生成模型-数据对比报告和可视化诊断图

**Uncertainty Quantifier / 不确定性量化器**
- 估计参数误差、置信区间和可信区域
- 处理观测噪声和系统误差的传播分析
- 提供模型选择准则（AIC、BIC、贝叶斯因子）
